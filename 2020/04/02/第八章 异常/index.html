Author: XinYS Date: April 2, 2020  23:23:19 Category: python

第八章 异常

  • 相比使用if/else语句，检查一个程序是否会出现异常的最好办法就是直接运行它，观察出现的异常情况
  • 并且异常处理不会导致代码可读性的下降

异常对象

  • python用异常对象表示异常状态，遇到错误时引发异常，异常对象未被处理（被捕获）时，程序终止并显示错误信息(Traceback)
  • 每个异常对象都是某个类的实例
类 名 描 述
Exception 几乎所有的异常类都是从它派生而来的
AttributeError 引用属性或给它赋值失败时引发
OSError 操作系统不能执行指定的任务（如打开文件）时引发，有多个子类
IndexError 使用序列中不存在的索引时引发，为LookupError的子类
KeyError 使用映射中不存在的键时引发，为LookupError的子类
NameError 找不到名称（变量）时引发
SyntaxError 代码不正确时引发
TypeError 将内置操作或函数用于类型不正确的对象时引发
ValueError 将内置操作或函数用于这样的对象时引发：其类型正确但包含的值不合适
ZeroDivisionError 在除法或求模运算的第二个参数为零时引发

raise

  • 用以引发异常，将一个类（必须为Exception的子类）或实例作为参数
  • 将类作为参数时，将自动创建一个实例

自定义异常类

  • 创建异常类必须直接或间接地继承Exception

    8.3 捕获异常

  • 捕获异常，即对异常进行处理
  • try / except语句，处理方式可以是打印自定义的错误语句，也可以重新引发异常（例如下面的raise）
  • 关于异常的顺序：若未处理在函数中出现的异常，将向上传播到调用函数的地方；仍未得到处理，将继续传播到主程序（全局作用域），若主程序也没有处理异常，则程序终止并显示栈追踪消息

不提供参数

  • 若无法处理异常，选择使用不带参数的raise，将重新引发它（即继续向上传播）

抑制异常

  • 抑制异常时，异常情况会隐式地返回None，适用于与用户的交互会话
  • 将执行对异常进行处理的语句（打印错误消息等），不让异常继续传播（即程序继续正常执行）

异常上下文

  • 若想引发别的异常，可以将异常作为raise的参数
  • 此种情况下，进入except子句的异常会作为异常上下文出现在最终的错误消息中
  • 可以使用raise … from … 来提供上下文，也可以使用None来禁用上下文（此时将显示最终引发的异常）

多个异常

  • 可以使用多个except子句或将多个异常写在一个元组中
  • 意义：若仅指定某种异常，那么将不会捕获除此类异常之外的异常以进行处理，从而会导致程序终止

一网打尽

  • 很难预测到所有可能出现的问题，程序会出错
  • 这样的意义在于，如果没有预测到这类异常情况的出现，也没有采取相应的措施，那么与其使用并非要捕获这些异常的try/except语句将其隐藏，不如让程序直接崩溃

else / finally

  • else 在没有出现异常时执行代码块
  • finally 是否引发异常都会执行，常用作清理代码
